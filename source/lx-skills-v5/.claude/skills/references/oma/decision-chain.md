# 裁决链 (Decision Chain)

> 所有 OMA skill 共享的 5 关裁决流水线。引用：`@reference/oma/decision-chain.md`

## 五关裁决

```
哲学关 → 铁律关 → 现状关 → Oracle Agent → Meta-Oracle → 人类审批
```

| 关 | 审查者 | 检查内容 | 不通过 |
|----|--------|---------|--------|
| **哲学关** | AI 自检 | 方案是否符合 7 条哲学优先级？ | 重生成方案 |
| **铁律关** | Hook 系统 | 是否违反 8 条铁律？ | BLOCKED |
| **现状关** | AI 自检 | 与现有代码/约定/机制是否一致？ | 调整方案 |
| **Oracle Agent** | 独立 agent | 源码级静态审核（P0/P1 问题） | REVISE → 修复后重审 |
| **Meta-Oracle** | opus critic（独立上下文） | 运行时验证 + 对抗性审查 + 盲区扫描 | ADVISORY/REJECT |
| **人类审批** | 人类 | 最终决策（仅不可逆/高风险操作） | 返回重新生成 |

## 裁决协议

### 哲学关（AI 自检 — 前置）

方案生成后，按哲学优先级逐条检查：

```
#4(没验证=没做) → 方案包含验证步骤？
#6(0信任) → 有独立审查环节？
#3(先守护) → 有回滚/熔断？
#7(文档优先) → 方案文档化？
#5(以人为本) → 方向指引 + 选项？
#2(少量大增益) → 方案是否最小化？
#1(渐进披露) → 信息按需加载？
```

任一不满足 → 标记缺口，补充后重检。

### Oracle Agent（静态分析）

- **模型**：critic/architect agent（独立 agent）
- **方法**：源码级审核 — 文件存在性、引用正确性、契约一致性
- **裁决**：
  - ACCEPT → 进入下一关
  - REVISE → 修复 P0 问题，重新提交（最多 3 轮）
  - REJECT → 停止，向用户报告

### Meta-Oracle（运行时 + 对抗性）

- **模型**：opus critic（独立上下文，不共享主会话）
- **方法**：运行时验证 > 静态检查，烟雾日志 > 文件存在性
- **触发**：G1（架构决策）/ G2（PRD 最后一步）/ G3（Oracle ACCEPT + ≥8.5）/ G4（Release）
- **裁决**：
  - ACCEPT → 继续
  - ADVISORY → 建议修正，AI 自行判断
  - REJECT → 强烈阻断，AI 需书面理由覆写；连续 2 次 REJECT = 事实阻断

## 裁决冲突优先级

```
Meta-Oracle REJECT > Oracle REJECT > 铁律 > 哲学裁决 > 现状约定
```

> 铁律关由 hook 系统强制执行（非 AI 自检），违规直接 BLOCKED。
