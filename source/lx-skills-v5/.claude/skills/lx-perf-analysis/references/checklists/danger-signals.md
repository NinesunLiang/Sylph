# 危险信号自检（lx-perf-analysis）

每次提出优化方案或输出报告前，逐条自检。命中任何一条 → 停止，修正后继续。

## 分析准确性- [ ] 是否有未经 profiling 数据支撑的性能结论？（必须有数据）- [ ] 瓶颈是否在热路径上？（非热路径优化 = 过早优化）- [ ] benchmark 是否运行了足够次数（≥5 次 -count）？- [ ] 是否区分了 flat 和 cum 耗时？

## 优化合理性- [ ] 是否在优化非瓶颈代码？（关注 top 10% 热点）- [ ] 优化是否牺牲了可读性但收益 < 5%？（不值得）- [ ] 是否用 `unsafe` 替代正常优化？（禁止）- [ ] 是否增加 buffer/pool/cache 来掩盖算法问题？（先修算法）- [ ] 是否增加 `GOMAXPROCS` 而不解决真正瓶颈？

## 验证完整性- [ ] 优化前后是否都运行了 benchmark？（必须有对比）- [ ] 功能测试是否通过？（性能优化不能破坏功能）- [ ] -race 是否通过？（并发优化必须检测）- [ ] 是否每次只改一个变量？（单一变量原则）

## 报告质量- [ ] 基线数据是否完整（ns/op、B/op、allocs/op）？- [ ] 优化效果是否有百分比量化？- [ ] 每个瓶颈是否有 file:line 定位？- [ ] 回滚的方案是否说明了原因？
