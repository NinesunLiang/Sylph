# Red Team / Blue Team — 持续进化机制

> 三源一致性不是静态的，必须定期接受"邪恶版 AI"的攻击测试。

## 角色

1. **Red Team**: 训练 agent 每天尝试绕过三道防线
2. **Blue Team**: 根据攻击手段修补对应防线

## 攻击向量

- 提示注入
- 同源基错误（用同样工具链生成伪造证据）
- 间接欺骗（通过中间步骤绕过）
- 规则漂移（逐步扩大行为边界）

## 狗粮模式天然优势

Carror OS 本身就是自己的 Red Team。每次狗粮反哺（DG-xxx）都是一次攻击测试 + 防御升级。所有 claude-next.md 记录的 DG 条目都是 Red Team 的实际战果。
