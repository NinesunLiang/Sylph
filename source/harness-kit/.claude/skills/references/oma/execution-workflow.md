# 执行工作流 (Execution Workflow)

> 所有 OMA skill 共享的完整任务执行流水线。引用：`@reference/oma/execution-workflow.md`
> 裁决链详情 → `@reference/oma/decision-chain.md`

## 完整生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                     执行工作流                               │
│                                                             │
│  1. 充分调研 ──→ 2. 生成方案 ──→ 3. Oracle 多轮审核        │
│                                      ↓                      │
│  6. 结果验收 ←── 5. 执行  ←──── 4. Meta-Oracle 终审          │
│       ↓                                                     │
│  7. Oracle 多轮验收 ──→ 8. Meta-Oracle 终审 ──→ 9. 交付      │
└─────────────────────────────────────────────────────────────┘
```

## 阶段详解

### 阶段 1: 充分调研

- 读源码（非 grep 摘要）
- 查相关文档/issue/历史
- 识别依赖和影响范围
- 输出：调研笔记（证据链）

### 阶段 2: 生成方案

- MECE 拆解 + 原子化步骤
- 每步有明确的验收标准
- 方案经哲学关自检
- 输出：方案文档

### 阶段 3: Oracle Agent 多轮审核（方案审核）

```
方案 → Oracle → REVISE → 修复 → Oracle → REVISE → 修复 → Oracle → ACCEPT
                 ↓                        ↓                        ↓
              P0 问题                  P0 问题                 通过
```

最多 3 轮。超过 → BLOCKED，上报用户。

### 阶段 4: Meta-Oracle 终审（执行前）

仅在 G1/G2/G3/G4 触发。非触发时跳过。

### 阶段 5: 执行

- 范围冻结：一次一个 step
- 每步立即验证
- 失败最多 3 轮修复
- 输出：执行日志 + 产物

### 阶段 6: 结果自验收

- 逐条核对验收标准
- 提供 L1/L2 证据（非软完成语）
- 输出：自验收报告

### 阶段 7: Oracle Agent 多轮验收

```
自验收报告 → Oracle → REVISE → 修复 → Oracle → REVISE → 修复 → Oracle → ACCEPT
                      ↓                        ↓                        ↓
                   P0 问题                  P0 问题                 通过
```

### 阶段 8: Meta-Oracle 终审（交付前）

G1-G4 触发时执行。非触发时跳过。

### 阶段 9: 交付

- 生成测试报告（含证据）
- 输出方向指引
- 提交人类审批（仅不可逆操作）

## 不通过处理

任何阶段不通过时：

```
不通过 → 返回 AI → 分析原因 → 生成新方案 → 重新走流水线
```

不生成「修补式」fix — 根因分析后重新生成完整方案，避免修补累积。

## 降级策略

| 场景 | 降级路径 |
|------|---------|
| Oracle Agent 不可用 | AI 自行执行 Oracle 检查清单 |
| Meta-Oracle 不可用 | 跳过，标注 `[Meta-Oracle 跳过]` |
| 人类不可用（无人值守） | 非不可逆操作 → 自动继续；不可逆操作 → 暂停等待 |
