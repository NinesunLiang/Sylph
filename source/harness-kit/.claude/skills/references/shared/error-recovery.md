# 共享错误恢复模式

> 通用错误场景 + 恢复路径。各 skill 继承此表，追加 domain-specific 场景。

## 通用场景

| 场景 | 检测 | 回退 | 重入点 | 升级条件 |
|------|------|------|--------|---------|
| 脚本不存在 | exit code ≠ 0 | 手动执行或跳过 | — | 报告缺失 |
| 文件损坏/格式异常 | readFile 失败 | 从 git 恢复或重建 | 重读 | 无法恢复 → 询问用户 |
| git 不可用 | git status 失败 | 跳过 commit，手动记录 | — | 提示手动操作 |
| 依赖缺失 | 编译/安装失败 | 自动安装或降级 | 重编译 | 需管理员 → skip-risk |
| 超时 | 操作超过配置时限 | 增加超时或拆分 | 继续当前 step | 3 次超时 → 升级 |
| 锁竞争 | 文件锁存在 | 等待释放或强制清理 | 重试获取 | 3 次失败 → 手动 |

## 修复上限（硬规则）

```
同一问题修复 ≤ 3 轮：
  第 1 轮 → 修复
  第 2 轮 → 换假设 + 修复
  第 3 轮 → 最后一次尝试 + 换假设
  仍失败 → BLOCKED → 升级用户

轻量模式（lx-todo）上限 2 轮，不可第 3 次尝试。
```
