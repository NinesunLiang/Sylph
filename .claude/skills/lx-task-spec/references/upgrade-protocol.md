## 升级协议（Todo → 主分支）

**三个硬性升级触发点**（不可跳过，不可增加重试次数）：
| 触发点 | 条件 | 检测方式|
|--------|------|---------|
|Step 1 分拣 | >3 文件 或 需设计 | `grep -l` 文件计数|
|Step 2 执行 | 🐛 debug 2 轮未定位 | 轮次计数器|
|Step 3 验证 | 2 次 blocked | 重试计数器 |
**升级执行**（必须完整执行，不可省略；根因分析遵循 `@../../nodes/execute_node.md`）：

```1
. 更新 .omc/state/todo-queue.md：将该项从当前区移到"已升级到主分支"，添加 reason + upgraded 日期2. 若有代码变更 → git stash（保留但不提交）3. 输出升级报告（传递给主分支的上下文）
1. 更新 .omc/state/todo-queue.md：将该项从当前区移到"已升级到主分支"，添加 reason + upgraded 日期2. 若有代码变更 → git stash（保留但不提交）3. 输出升级报告（传递给主分支的上下文）
```

**升级输出模板**：

```↑ Todo #[id] 升级到主分支
类型：[🐛/✨/🔧]描述：[原始描述]升级原因：[>3 文件 / debug 2 轮未定位 / 验证 2 次 blocked]已有进展： - 代码变更：[git stash ref 或 "无"] - 调试发现：[已排除假说 / 已收集证据 / "无"] - 门禁报告：[lx-pre-commit 输出摘要 / "未执行"]建议入口：[3] Coding / [4] TDD / [4a] Debug
```
