# Feature PRD 脚手架模板

```bash
mkdir -p prd/{sub_prd_name}/feat-XXX/{state,contracts,mocks}
cat > prd/{sub_prd_name}/feat-XXX/prd.md << 'EOF'
# Feature: {Feature 名称}

> 所属 Sub PRD：{Sub PRD 名称}
> 职责：{一句话描述}

## 功能边界

- **负责**：
  - {从 Sub PRD 职责中提取}
- **不负责**：
  - {明确不属于本 Feature 的职责}

## 对外接口

| 接口 | 方向 | 入参 | 出参 |
|------|------|------|------|
| {接口名} | {inbound/outbound} | {入参} | {出参} |

### 事件（如适用）

| 事件名 | 方向 | 说明 | 载荷 |
|--------|------|------|------|

## 非功能要求

| 属性 | 约束值 | 优先级 |
|------|--------|--------|
| {指标} | {约束} | {P0/P1/P2} |

## 数据实体归属

| 实体名 | 归属关系 | 操作类型 | 说明 |
|--------|---------|---------|------|
| {实体} | Own/Read/Write | {CRUD} | {说明} |

## 依赖关系

- **依赖**：{本 Feature 依赖的其他模块}
- **被依赖**：{哪些模块依赖本 Feature}

## Mock 数据

```json
{
  "{场景名}": { "input": {}, "output": {} }
}
```

## 验收条件

- [ ] AC-1: {描述}
- [ ] AC-2: {描述}

## 技术约束

- {约束 1}
EOF
```
