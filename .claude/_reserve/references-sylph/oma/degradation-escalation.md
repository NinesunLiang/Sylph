# 降级升级策略

> OMA 全系列 skill 共享的降级框架。每个 skill 的「降级策略」表是此框架的实例化。

## 降级层级

```text
主路径（最优）→ 降级路径（次优但可行）→ 人工介入（阻塞）
```

## 降级时机

| 触发条件 | 动作 |
|---------|------|
| 脚本/工具不可用 | 降级为手动替代方案 |
| 外部依赖超时 | 跳过该步骤，标注缺失项 |
| 3 轮修复未通过 | 标记需人工介入，不阻塞其他任务 |
| Oracle 不可用 | 降级为 AI 自审，标注 `[无Oracle审核]` |
| Phase 0 不可用 | 跳过前置澄清，标注风险 |

## 通用降级表模板

每个 skill 的降级策略表应覆盖：

| 场景 | 主路径 | 降级 |
|------|--------|------|
| 工具/脚本缺失 | 自动化校验 | 手动自检清单 |
| 上游 skill 不可用 | 委托调用 | 手动执行对应命令 |
| 外部服务超时 | 正常流程 | 跳过 + 标注缺失 |
| N 轮修复失败 | 自动修复 | 标记需人工介入 |
| Gate/Oracle 不可用 | 门禁审核 | 标注 `[无Oracle审核]` 继续 |

## 降级记录格式

```text
[DEGRADE] {场景}：{主路径} 不可用 → 降级为 {降级路径}。影响：{评估}
```
