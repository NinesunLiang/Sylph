# 危险信号自检（lx-code-review）

每次生成审查报告前，逐条自检。命中任何一条 → 停止，修正后继续。

## 审查准确性- [ ] 是否有未读取上下文就报告的问题？（必须读 ±10 行）- [ ] 是否把 goctl 生成代码标记为问题？（应排除）- [ ] 是否把合理的 `//nolint` 标记为问题？（检查理由）- [ ] 是否在不确定时以确定语气报告？（应标记"待确认"）

## 误报防范- [ ] error 被吞：是否检查了上下文（可能有 defer 处理）？- [ ] go-zero 层违规：是否确认了项目确实使用 go-zero？- [ ] 性能反模式：代码是否在热路径中？（非热路径不报 P1）- [ ] 接口过大：是否为外部库的接口实现？（不报告）- [ ] 错误路径无日志：是否为最底层 model 层（由上层统一记录）？- [ ] context 未传日志：是否在 main/init 阶段（无 ctx 可传）？- [ ] 关键操作无日志：框架是否已自动记录（go-zero middleware）？- [ ] metric 缺失：go-zero stat middleware 是否已启用？

## 建议质量- [ ] 修复建议是否遵循项目已有惯例？- [ ] 代码示例是否能直接编译通过？- [ ] 是否给出了"为什么是问题"的解释？- [ ] P0 问题是否给出了可直接复制的修复代码？

## 报告完整性- [ ] 严重度计数是否与详情匹配？- [ ] 每个问题是否有 file:line 定位？- [ ] 待确认项是否单独标注？
