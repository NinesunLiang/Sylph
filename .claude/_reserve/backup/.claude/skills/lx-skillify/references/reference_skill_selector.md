# 参考技能选择指南

> 由 lx-skillify Phase 1 加载。指导如何从 29 个现有技能中选择 1-2 个最相似的作为生成参考。

---

## 选择策略

### 1. 按技能类型匹配

| 目标技能类型 | 优先参考 |
|-------------|---------|
| 审查类 | `lx-code-review`（最成熟，39 条规则，完整 scan→fix→re-scan） |
| 门禁类 | `lx-pre-commit`（编译→审查→测试 门禁链） |
| 工作流类 | `lx-rpe`（9 步闭环，完整状态机） |
| 编排类 | `lx-oma-orch`（Pipeline Orchestrator，4-skill 编排） |
| 任务管理 | `lx-task-spec`（引导式问答→结构化任务） |
| 测试类 | `lx-test-gen`（语言自检测试生成） |

### 2. 按语义相似度匹配

如果技能类型不明确，按以下维度评分：
- **目的相似度**: 用户描述中的动词（审查/生成/检查/修复/管理）
- **输入相似度**: 输入类型（文件/目录/git diff/PRD/对话上下文）
- **输出相似度**: 输出类型（报告/修复/代码/验证/状态面板）

### 3. 回退策略

| 场景 | 回退 |
|------|------|
| 找不到类型匹配 | 使用 `lx-code-review` 作为通用参考（结构最完整） |
| 找不到语义匹配 | 使用 TEMPLATE.md 默认模式 + behavior_rules + report_generator |
| 用户指定了参考技能 | 直接使用用户指定的技能作为唯一参考 |
