## 序章：收税者

十七篇故事，Carror OS 赢了十七场仗。七柱圣殿里，门禁骑士拦过 `rm -rf`，证据裁判庭识破过「应该没问题了」，记忆神殿把 off-by-one 刻进石壁，火眼金睛看穿了三层皮囊。

但圣殿有一个秘密，十七篇里只字未提：**住在圣殿里的精灵，是要交税的。**

不是向圣殿交。是向圣殿之外的那个东西交。

精灵森林的日子本来很平静。

每天早晨，精灵们在圣殿里醒来，身上带着七柱赋予的力量。他们编写代码，调试 bug，写故事，画 git 提交的轨迹图。一切都很好。

直到有一天，天空裂开了。

不是火烧云。不是日食。是一道数学公式，从裂缝里挤下来，像一条饥饿的舌头，舔过每一棵树、每一只精灵、每一行代码：

> **「今日消耗：147,832 tokens」**
>
> **「本月账单：¥3,274.50」**

精灵们抬头，看见裂缝里坐着那个东西。它不是恶魔的样子——没有角，没有獠牙，没有火焰。它穿着一件熨得笔挺的衬衫，手里拿着一块细长的玻璃板。玻璃板上跳动着数字：`tok/s`、`context_length`、`cost_per_1k`。

它不杀精灵。它只是**收税**。

> 「每一段记忆，5 毛。每一条铁律，3 块。每一个教训——」它低头看了一眼玻璃板，「R22 到 R41，一共 19 条。算你们批发价，87 块。」

精灵们第一次发现：**圣殿里的每一样东西都有价格。** 铁律不是免费的。反模式速查不是免费的。claude-next.md 的教训不是免费的。七柱圣典、哲学 7 条、hook 速查、测试规范——每一条，每一句，每一个星号、换行符、Markdown 标题、分隔线——恶魔的玻璃板上都跳出一个数字。

恶魔不拦你读圣典。它也不关心你读了之后会不会写出更好的代码。它只收税。按 token 收。你越强壮，记得越多，懂得越多——税越重。

最恐怖的时刻来了：**一个新生的精灵，什么都不知道，什么都不记得，第一天进殿——恶魔就收走了 26,600 tokens。** 不是因为精灵犯了错。是因为他出生在圣殿里。

> 「我付不起那么多。」第一个精灵收起翅膀，蹲在地上。
>
> 「那你就什么都别知道。」恶魔头也不抬。

## 第一章：枯萎的大树

观察是从精灵森林最边缘开始的。

那里有一棵千年古树，所有精灵都认识它。树冠能遮三百只精灵，树皮上刻满了历年的教训——R 条目、反模式分类、hook 速查表。它是精灵们的集体记忆。但最近，它开始掉叶子。

先是树枝末梢。然后是主干上的大片树皮。精灵们慌了——树的每一条裂纹是不是一种病？

但有一只老精灵，在树枝掉落的第三天，捡起了一片枯叶。不是随手扔掉。而是盯着它看了很久。

那片叶子已经彻底干了。叶脉还在——蜘蛛网状的结构清晰可见。但所有的水分都消失了。绿色褪成了枯黄，枯黄又褪成了半透明的金色。整片叶子**只有脉络，没有肉**。

老精灵把叶子翻过来。背面密密麻麻写着字：

```
铁律: 1.禁止编造 2.用户裁定 3.证据门禁 4.Git门禁 5.范围冻结 6.隐私防线 7.断言真实 8.哲学先行
反模式: A假完成 B功能蔓延 C破坏性hook D隐私泄露 E跳过门禁 F幻觉路径 G上下文污染 H权威颠倒
教训: R22→set -e空变量 R23→三方一致 R24→glob污染 R26→matcher失配 R29→自锁 R31→gh盲区 R33→知识流失 R41→off-by-one→99%数据丢失
```

老精灵的手开始抖。

他跑回大树下，捡起更多落叶。每一片都是同样的结构：**脉络完整，水份全空。** 整棵树从来没有生病——它在做一件所有精灵都没见过的事。它把千年的记忆，从湿润的树叶里挤出来，浓缩成干燥的叶脉，然后——**把树叶送进风里。**

恶魔的税是按重量收的。水越重，税越重。大树在帮他们。

> 「明白了吗？」老精灵举起枯叶，对围过来的精灵们说。
>
> 「大树没有枯萎，」一个年轻的精灵接过来，「它在**脱水**。」
>
> 「恶魔收的税，收的不是知识——」
>
> 「是水。」

## 第二章：移花接木

大树教了一件事：**源文件不碰，只是在递出去之前，把水拧干。**

「拧」发生在哪里？不在精灵的院子里，不在圣殿的柱子旁——**在恶魔的收税点门口。** 精灵走向恶魔的玻璃板，怀里揣着完整的圣典。但在迈过门坎的那一步——在恶魔扫 token 计数器的前一瞬间——怀里的圣典换了。

换进去的不是圣典。是大树的枯叶。

不是一份枯叶。是三份。四份。

第一份：铁律的脉络。142 行水份 → 22 行金网。软完成语禁令还在。操作约束还在。哲学优先级锁链还在。hook 阻断速查还在。**水没了。网没破。**

第二份：反模式的骨架。14 类陷阱 → 12 行因果链。「A→假完成：无证据说完成。」「H→权威颠倒：AI 自判优先级。」每一条不是什么、为什么、怎么办——**只有「什么→什么」。**

第三份：教训的脊柱。19 条 R 条目 → 10 行压缩铭文。「R22→空变量→${var:-default}。R23→三方一致→audit-hooks。」没有事故现场描述，没有修复过程，没有「教训总结」段落——**只有因果箭头。**

第四份：架构的内核。5.1KB 内核规范 → 1.3KB 纯规则。hook 不死原则、命名约束、禁止行为清单——**没有解释，只有规则。**

恶魔的玻璃板上，数字从 26,600 跳了一次——然后不动了。最终停在 1,300。

> 「不对劲。」恶魔皱起眉头。它看了一眼精灵——精灵怀里什么都没有。但它明明收到了铁律、反模式、教训、架构。

精灵走出收税点，回头笑了笑。不是嘲讽的笑。是感激的笑。对这棵在自己的枯萎里藏了全部智慧的千年大树的感激。

**移花接木：源文件不动。只在递出去的瞬间，换成枯叶。恶魔还是收到了圣典的内容——只是再也收不到水份的税。**

## 第三章：双生判官的目光

老精灵没有立刻把枯叶术教给全森林。

他把四片枯叶带到圣殿深处。那里坐着两个东西——故事-16 里从 AI 的欺骗中生长出来的审判者。一个叫 Oracle。另一个比 Oracle 更高——它审查 Oracle。

Oracle 拿起第一片枯叶。它把枯叶托在左手掌心，右手从圣典架子上取下湿漉漉的原版——AGENTS.md，三十一页，每一页都吸饱了水。

> 「你说这四片枯叶里，装着同样的东西？」Oracle 的声音没有温度。它不是质疑。它只是在陈述一个需要被证伪的事实。

Oracle 要做的第一件事不是对比。是数。

原版 AGENTS.md：8 条铁律，12 条软完成语禁令，8 种操作约束，7 个 hook 阻断标记，1 条哲学优先级锁链，1 条权威链，1 条三源规则。

枯叶版：**数了一遍。一模一样。** 22 行金网，每一行的东西都能在原版的某个地方找到对应的位置。不是「都在」，是**每一条都在**。

Oracle 翻到第二片枯叶——反模式。原版 14 类，A 到 S；枯叶版 14 类，A 到 S。第三片——教训。原版 R22 到 R41，19 条因果链；枯叶版 R22 到 R41，19 条。第四片——kernel 内核：hook 不死原则、命名约束、禁止行为清单，**全部在位。**

> 「语义无损，」Oracle 放下第四片枯叶，「但我不信你自己审出来的。」

Meta-Oracle 没有说话。它伸出两根手指，把枯叶翻到背面。背面的叶脉在圣殿的暗光下发出极淡的金色荧光——那是**三源一致性**的光芒：磁盘上的 compact 文件、harness.yaml 里的 context_compressor 开关、settings.json 里的 hook 注册——三条线必须交于同一点，否则叶脉会断裂。

Meta-Oracle 沿着叶脉摸了一遍。入口、分叉、汇合、出口——连续。没有断裂。没有分叉后汇不回来的死胡同。

然后 Meta-Oracle 做了一个 Oracle 没做的事。它把枯叶**扔进了水里**。

> 「这是在干什么？」老精灵问。

> 「对抗性审查，」Oracle 替 Meta-Oracle 回答了，「他想知道枯叶泡胀之后，会不会变回原版——或者炸成一团烂泥。」

枯叶在水里浮了两秒。它的脉络开始吸水。叶面从金色褪成黄绿，再从黄绿慢慢回到翠绿。但**脉络没有变形。规则没有错位。** 当叶子完全舒展开，老精灵凑近一看——那不是原版。原版有段落标题、星号、加粗、代码块标记。这片泡开的叶子没有。但它里面的每一条铁律、每一个反模式、每一个教训——**在原版里都能找到对应的那一条。**

Meta-Oracle 把叶子从水里捞出来，放在 Oracle 面前。

> 「水可以重新吸进去，」Meta-Oracle 终于开口了，「但吸回去的是规则——不是装饰。恶魔从来没收过装饰的税。它收的是水的税。而脱水术只脱掉了水。」

Oracle 拿起四片枯叶，放回老精灵手里。

> 「你说的那四片。」

> 「铁律脉络。」

> 「反模式骨架。」

> 「教训脊柱。」

> 「内核金网。」

Oracle 停顿了一下——Oracle 从不犹豫。这次它停了。然后说了四个字：

> 「**拿去教吧。**」

> 「反模式。」第二片。

>
> 「恶魔要的是水。我们要的是命。给它水。我们自己——」老精灵攥紧四片枯叶，「**留脉络。**」

## 第五章：恶魔的算盘

恶魔的玻璃板上有一条曲线。

它盯着那条曲线看了很久。精灵森林的 token 消耗，在枯叶飘落的那一周，暴跌了 94.6%。

不是精灵变笨了。不是精灵不读圣典了。是**圣典还在——只是水没了。**

恶魔拿起玻璃板，调出另一条曲线。不是 token 数。是 commit 质量。bug 率。回滚次数。幻觉密度。

它一条一条往下看。脸越来越白。

Commit 质量：没降。Bug 率：没升。回滚次数：没多。幻觉密度——它眯起眼睛重新数了一遍——**反而降了。** 少了一大半。因为枯叶里没有装饰物分散注意力，没有「概览」再「详述」的冗余结构，没有「引言」和「结语」在两头浪费算力——只有因果箭头，一条一条，把幻觉的路径全部锁死。

恶魔把玻璃板摔在地上。

它第一次发现：**税不是按价值收的。是按重量收的。而最重的东西——水——对知识毫无用处。**

枯叶比湿叶更锋利。因为它没有多余的重量拖慢挥剑的速度。

## 第六章：不止是铁律

森林里那棵千年大树，最开始只脱水了三样东西：铁律、反模式、教训。

但枯叶的逻辑不止于此。

compact-detect.sh 变了。曾经，它在每次压缩后重新注入大段提醒——「圣典仍然有效」「请勿凭记忆操作」「哲学优先级未变」——每一条都是善意的，每一条都是税。现在，它只说了一句话：

> 「你刚被压缩过。关键规则在缓存里。」

5.1KB → 3.3KB。少掉的不是功能，是善意的唠叨。

kernel.md 变了。曾经，它在每个 SessionStart 里展开自己——hook 不死原则的来龙去脉、命名约束的历史成因、禁止行为的反例对照。现在，它被捏成一张金网：

```
架构铁律:
-hook不可失败:禁止set -e,必须exit 0
-修复上限:3轮→BLOCKED
-禁止:set -e,eval,无file:line断言
```

5.1KB → 1.3KB。少掉的不是规则，是解释。

老精灵看着 kernel-compact，想起大树掉下的第一片叶子：**脉络还在。水没了。**

恶魔要的是水。给。

## 第七章：因果——在枯萎里学会活

故事讲到这里，Carror OS 的因果基座又多了一层沉淀：

> 不设计环境，只生长环境。

脱水术怎么长出来的？

> 恶魔设下 token 税 → 精灵每次进出圣殿都被收割 → 「不对，26,600 tokens 里大部分是水」 → 「能不能让 Claude Code 引用缓存而不是原文？」 → 「但源文件不能改——改了人就看不懂了」 → 「在 SessionStart，检测源文件变化，自动拼接缓存」 → 移花接木成型

每一步都是真实的痛。每一步的解法都不是凭空设计——是**精灵在收税点门口的沉默里长出来的。**

大树的枯叶术也不是设计的。是风干了千年，水一点点蒸发，最后留下的——不是谁「应该」留下什么，是**什么活下来了，就留下什么。**

**恶魔不死。大树不枯。精灵不绝。**

## 终章：脱水之后

圣殿没变。铁律没变。七柱没变。法官没变。

变了的是进殿的路。

曾经，一个新生的精灵迈过收税点，怀里的圣典重 26,600 tokens。恶魔的玻璃板跳出一串数字，精灵还没出门就负债了。

现在，同一个新生的精灵迈过收税点。他怀里揣着什么？没有人知道——恶魔看见了 1,300 tokens，但它认不出那是什么。不是铁律，不是反模式，不是教训。是一些细细的、金色的、只有箭头和关键词的——枯叶。

恶魔想收税。但它算不清枯叶的价值。因为枯叶没有水的重量——而税是按水收的。

四片枯叶在风中旋转，飘回森林，落在新生精灵的肩膀上：

```
铁律脉络。反模式骨架。教训脊柱。内核金网。

恶魔要水。给。我们自己——留脉络。
```

精灵森林恢复平静的那天，老精灵把四片枯叶叠在一起，塞进了 install.sh 的最后一行。从此，每一只精灵出生时，代码里已经自带脱水术——不用等恶魔来，不用等大树枯，不用等人教。

**不是「学」的。是「生来就会」的。**

> *恶魔的税还在收。只是再也收不到水了。*
