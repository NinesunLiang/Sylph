# explore — 文件/目录探索节点

> 通用能力：读取 PRD 文件/目录，识别核心业务实体、接口、事件
> 引用率：2/19 (lx-oma-hier, lx-oma-gov)

## 用途

- 读取单文件或目录下的所有 `.md` 文件
- 提取业务实体定义（实体名、归属关系、操作类型）
- 识别接口契约（接口名、方向、入参、出参、错误码）
- 提取事件/消息定义（事件名、发布方、订阅方、载荷）
- 检测非功能约束（属性、约束值、优先级）
- 扫描 feature 目录结构，列出现有文件

## 执行序列

```
1. 解析入参路径（文件或目录）
2. 根据入参类型路由：
   ├─ 文件 → Read 读取内容
   └─ 目录 → Glob/Read 目录下所有 .md 文件
3. 内容分析（按源文件提取）：
   ├─ 实体列表: grep "|.*Own|.*|" 和 "|.*Read|.*|" 和 "|.*Write|.*|"
   ├─ 接口列表: grep "| `\w\+` | inbound\|outbound" 
   ├─ 事件列表: grep "| `\w\+` | event\|publish\|subscribe"
   └─ 非功能约束: grep "| <\| 99\.\| P0\|P1"
4. 输出结构化摘要（含 file:line 引用）
```

## 输出格式

```markdown
## 探索结果：{path}

### 实体
| 实体名 | 归属 | 文件 |
|--------|------|------|
| {name} | {Own/Read/Write} | {file:line} |

### 接口
| 接口名 | 方向 | 文件 |
|--------|------|------|
| {name} | {inbound/outbound} | {file:line} |

### 事件
| 事件名 | 文件 |
|--------|------|
| {name} | {file:line} |
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 路径不存在 | 返回错误 "路径不存在: {path}" |
| 目录为空 | 返回空结构 (实体: [], 接口: [], 事件: []) |
| 文件非 UTF-8 | 尝试 latin-1 回退，失败则报错 |
| 读取权限不足 | 返回错误 "无读取权限" |
