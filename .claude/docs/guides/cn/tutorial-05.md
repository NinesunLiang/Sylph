# T05 — 自主执行

> 📍 怎么让它自己跑 | [← T04](tutorial-04.md) | 下一篇：[T06 深度治理](tutorial-06.md) →

安全网很好。但有时候你不想每一步都点"确认"。

## Ghost — 方向驱动的探索

给一个方向，AI 一步步探索。每轮只走一步，不启动并行 agent。适合"我不知道具体要什么，但我往这个方向走"。

```
/lx-ghost on "源码级阅读：分析所有 hook 的实现逻辑"
```

安全网降级：permission-gate 不再硬阻断，completion-gate 从阻断改为写日志，fuzzy-block 放行模糊指令（"继续"是合法的迭代命令）。

**Ghost 适合**：探索未知代码、理解不熟悉的模块。**不适合**：完成任务、交付产物。

## Goal — 目标驱动的全自动

给一个目标，AI 拆解 → 你确认一次 → 全自动执行到完。你可以去睡觉，第二天看报告。

```
/lx-goal on "清理所有 docs/ 下已过时的 v6.2.0 历史备忘录"
```

安全网同样降级。但所有"本该阻断"的事件都记录到日志。安全网从实时阻断改为事后审计，从不消失。

---

| | Ghost | Goal |
|---|-------|------|
| 驱动方式 | 方向 | 目标 |
| 执行方式 | 增量，每轮一步 | 全量，一次完成 |
| 用户交互 | 可随时注入方向调整 | 仅开始确认一次 |
| 适合 | 探索未知领域 | 明确交付任务 |
