# 执行工作流

> OMA 全系列 skill 共享的执行框架。定义自主执行模式下的核心约束和常见场景处理。

## 核心铁律

1. **不暂停** — 不等待人类输入
2. **不提问** — 歧义按决策框架判断
3. **不中断** — 卡点处理后继续
4. **只记录** — 风险和阻断写入 skipped_risks

## 执行模型

```text
输入 → 参数处理 → 核心逻辑（按阶段推进）→ 校验 → 输出报告
                              ↓
                    卡点 → 按裁决链处理 → 继续/跳过
```

## 阶段式执行模式

多数 OMA skill 遵循以下流程模式：

```text
Phase 0: 前置澄清 → 消除不确定项 → 激活
Phase 1→N: 全自动执行 → 不暂停/不提问/不中断
退出: 结构化报告 → 已完/已跳/需人类
```

## 常见场景自主处理

| 场景 | 自主处理 |
|------|---------|
| 修复范围超预期 | 评估仍在目标内→继续，否则 skip-risk |
| 需安装依赖 | 能自动装则装，需管理员权限→skip-risk |
| 远程推送 | commit 照常，push→走裁决链 |
| Context Guard 阻断 | 创建 override 文件，改用 Bash |
| Permission Gate 拦截 | 走三级裁决链 |
| 发现无关问题 | 记入附带发现，不偏离主线 |
| 子任务冲突 | Philosophy #2 选择更高价值路径 |
| 硬边界触发 | 立即跳过→hard-boundary-hit→继续其他 |

## 最大自主权

在硬边界之外，AI 拥有以下领域的完全自主权：

| 领域 | 自主权范围 |
|------|-----------|
| 文件创建/修改 | 所有非治理文件 |
| 代码重构 | 自主决定方案和范围 |
| 架构决策 | Philosophy→Iron Rules→自主判断 |
| 子 Agent 调度 | 自主决定并行/串行/race |
| 依赖安装 | 自主安装，需 sudo 则 skip-risk |
| 测试运行 | 自主执行全部测试 |
| Git 只读操作 | status/diff/log/blame/show |
| 模式降级 | 遇到阻塞时自主降级 |
