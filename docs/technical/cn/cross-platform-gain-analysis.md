五、关键洞察

洞察 1：OpenCode 的投资回报率最高

OpenCode 无治理基准分：48.88（满分100）
OpenCode 有治理得分：  77.20
增益：                 +28.32分（+57.9%提升）

原因：OpenCode 的原生基础设施几乎为零（无 hook、无 agent 体系、无持久化），Carror OS 的 prompt 层注入几乎等于"从零建立"治理能力，边际收益极大。
这意味着：把 Carror OS 规范写进 AGENTS.md 并在 OpenCode 中运行，是最低成本、最高增益的组合。

洞察 2：Claude Code 的 G 维度出现"治理副作用"（-0.30）

Claude Code G维度：
  无治理：17.5（OMC 基础设施天然支持）
  有治理：17.2
  Δ：    -0.30

这不是 bug，是 feature 冲突。具体体现在三个子项：

| 子项 | 冲突原因 |
|---|---|
| G4 模式沉淀 (-0.60) | OMC 的 learner skill 有自己的模式沉淀路径（写入 claude-next.md）；Carror OS 的 claude-next.md 格式约束与 OMC 格式存在竞争，AI 在两套规范之间选择时产生摩擦。 |
| G5 偏差自适应 (-0.42) | OMC 要求 Self-Improvement Loop 在用户纠正后更新 claude-next.md；Carror OS 也要求同样的行为，但两者的"什么值得记录"的筛选标准略有差异，导致写入内容冗余或冲突。 |
| G6 长期一致性 (-0.72) | OMC 的 ralph 和 autopilot 已有"The boulder never stops"的持续机制，Carror OS 的"上下文守卫 >40% 触发总结重置"与 OMC 的 continuation enforcement 在某些场景下产生方向冲突（一个要继续，一个要总结重置）。 |

结论：Claude Code 已有成熟的治理基础设施，Carror OS 的部分规则与 OMC 规范形成"双轨竞争"，需要做规范融合而非叠加。

洞察 3：E2 幻觉抑制是跨平台最稳定的增益来源

CC  E2 增益：+4.48（基准2.4→6.88）
OC  E2 增益：+4.72（基准2.0→6.72）
Codex 增益： +3.28（基准2.0→5.28）
Cursor增益： +2.80（基准2.0→4.80）

无治理时，所有平台的幻觉抑制基准分都在 2.0-2.4（满分 8）区间——这是模型的自然幻觉率决定的，不是平台原生能力能解决的问题。
Carror OS 的铁律 #1/#7 + 软语言禁令是目前已知的最有效的 prompt 层幻觉抑制手段。

洞察 4：C5 知识深度几乎不受 Carror OS 影响

知识深度增益（所有平台均约+0.3~0.5）：这是最小增益项。

原因：知识深度由底层模型决定，prompt 层治理框架无法让模型"知道更多"，只能让模型"更诚实地呈现已知"（这才是 C9 证据引用的价值）。

洞察 5：Codex ≈ Cursor 增益基本相同，但结构不同

Codex 增益 +18.54：E维度贡献 7.70，G维度贡献 5.94
Cursor 增益+18.06：E维度贡献 7.06，G维度贡献 5.96

两者 G 维度增益几乎相同（+5.94 vs +5.96），但 Codex 在 E 维度多出约 0.7 分——原因是 Codex 的沙盒执行对 E5（虚假完成检测）有额外贡献，Carror OS 的 prompt 层在沙盒环境上叠加效果更好。
