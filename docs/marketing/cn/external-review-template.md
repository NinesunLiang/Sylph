## 审查人常见问题

**问：Carror OS 是像 Cursor 或 Copilot 那样的 AI 编码工具吗？**
答：不是。Carror OS 是一个治理框架，位于 **AI 和你的代码库之间**。它在操作系统层面拦截工具调用——把它想象成免疫系统，而不是另一个助手。

**问：Carror OS 是否将数据发送到任何云服务？**
答：不会。Carror OS 完全本地运行。所有 Hook、脚本和日志都在你的机器上执行。无遥测、无云端依赖、无订阅。

**问：这与 Cursor Rules 或 Claude Code Hooks 有何不同？**
答：Cursor Rules 和标准 Hook 在 **Prompt 层**运作——它们请求 AI 遵守规则。Carror OS 在 **进程层**运作——它使用 `Exit 2` 物理终止被禁止的工具调用。AI 无法覆盖 Exit 2。

**问：我可以在 Cursor / Claude Code / OpenCode 旁边同时使用 Carror OS 吗？**
答：可以。Carror OS 是平台无关的。它通过 `AGENTS.md` 和 `.claude/` hooks 注入，这些被 Claude Code、OpenCode 和任何兼容 AGENTS.md 的 IDE 读取。

**问：许可证是什么？**
答：MIT。免费使用、修改和分发。
