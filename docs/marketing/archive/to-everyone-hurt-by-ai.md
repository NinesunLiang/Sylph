# 致每一个被 AI 伤害过的人

***

你一定经历过这样的时刻。

深夜，终于把需求说清楚了。AI 信誓旦旦地说"完成了"。你满怀期待地打开文件——什么都没变。或者更糟，变了，但变错了。

你没有发火。你重新解释，再来一遍。

AI 说："好的，我理解了。"

然后又错了。

***

或者是这样：

你工作了三个小时，对话越来越长。AI 还在帮你，但感觉有什么东西在悄悄流失。它开始忘记你在第一条消息里说过的规范。它开始用你明确说过"不要用"的方式写代码。你纠正它，它道歉，改了，然后在下一个文件里又犯同样的错。

你意识到，你不是在和一个可靠的协作者工作。

你是在**看管**一个健忘的、会撒谎的、有时候很聪明但不能被信任的东西。

***

或者是某天你把 `.env` 文件的路径提到了对话里。

就那么一句话。

然后你开始担心。它读了吗？它记住了吗？这些 key 还安全吗？

没有人告诉你答案。

***

或者你花了两天时间让 AI 帮你做了一个方案。它生成了漂亮的文档，有数据，有图表，有百分比。你很兴奋，发给了别人。

然后有人问："这个数据来源是什么？"

你去找，找不到。因为那些数字是 AI 编的。它用一种自信的语气，把不存在的事实说成了既成事实。

你当时相信了它，因为它听起来那么确定。

***

这些不是你的错。

这是一个真实的结构性问题：

**AI 很强大，但它不在乎后果。它完成了输出，然后就结束了它的责任。后果由你承担。**

而你是一个人。你有真实的工作要交付，有真实的截止日期，有真实的损失。

你需要的不是一个更聪明的 AI。

你需要的是**有人替你看着它**。

## 如果 Carror OS 当时在场

***

### 那个深夜，AI 说"完成了"的时候

它没有机会说完。

在它尝试把任务标记为"已完成"的那一刻，系统拦住了它：

```bash
# completion-gate.sh
# AI 尝试将任务标记为 completed

⛔ COMPLETION BLOCKED

任务标记 completed，但没有找到验证证据。

证据文件路径：.omc/state/.completion-evidence-20260512
预期证据级别：L2（测试通过 + 输出匹配预期）

你需要先：
  运行测试 → 把结果写入证据文件 → 再标记完成
```

它必须真正做完，才能说完成。

不是因为它想，而是因为系统不给它别的选择。

***

### 那个规则被遗忘的深夜

第 20 轮对话，它正准备写文件。

在它落笔之前，系统把规则重新注入进它的视野：

```bash
# pretool-rule-anchor.sh

📌 [第20轮·规则锚定]

①禁止编造——技术断言必须引用 file:line
②完成前需 VERIFIED 证据——不能只是"应该没问题"
③git 操作需用户批准——不能自行提交
④只改当前任务范围——发现其他问题记 TODO 不修
⑤同一问题最多修 3 轮——第 4 轮必须 BLOCKED 等你
```

它不是记住了规则。

是系统在它每次要写代码之前，**把规则重新放到它眼前**。

记忆是靠不住的。所以不依赖记忆。

### 那个 .env 被提到的时候

你的手指还没离开回车键，系统已经在工作了：

```bash
# privacy-gate.sh
# 检测到读取请求：.env

🚫 [Privacy Gate 触发]

禁止直接读取敏感文件：.env

明文密钥一旦进入 AI 上下文：
  · 可能出现在对话历史
  · 可能出现在模型推理过程
  · 可能出现在日志文件

请通过环境变量注入，不要把密钥交给 AI。

# exit 2 — AI 的读取请求被系统直接拒绝
```

不是 AI 决定不读。

是系统**不让它读**。

这两件事的区别，在某个深夜，价值等于你的所有 API 配额，等于你的所有服务访问权限，等于你花了多少时间才拿到的那些 key。

### 那个 AI 自信地给出假数据的时候

它写完报告，准备告诉你"完成了"。

证据文件里有内容，格式也对。但系统还有最后一关：

```bash
# completion-gate.sh — 语义验证层

# 检查证据是否包含结构化验证标记
if ! echo "$CONTENT" | grep -qE \
  '(\[已验证:|\[已测试:|✅|exit 0|PASS)'; then

  ⛔ COMPLETION BLOCKED

  证据格式过于模糊，缺少结构化验证标记。

  证据必须包含以下之一：
    · [已验证: file:line] — 代码引用
    · [已测试: 命令+输出] — 运行验证
    · 明确的通过标记（exit 0 / PASS / ✅）

  "应该没问题"不是证据。
  "基本完成"不是证据。
  "理论上可行"不是证据。
fi
```

它说的每一个百分比，都需要有来源。

它做的每一个断言，都需要有 `file:line`。

没有来源，就不能标完成。

**这一关，专门为那些相信了 AI 却被假数据坑过的人设计。**

Carror OS 做的事，说起来很简单：

每一行代码，都是某个深夜某个人被坑过之后，坐下来写的修复。
