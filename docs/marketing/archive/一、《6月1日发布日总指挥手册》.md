# Carror OS

## Why Carror OS

AI coding 最大的问题，不是模型不够聪明，而是：

- 会泄露敏感信息
- 会在长上下文里失控
- 会产生自证偏差
- 会绕过验收直接交付
- 会把“看起来能跑”伪装成“真的可交付”

Carror OS 的目标不是替代你的开发环境，
而是为 AI 编程增加一层**操作系统级治理**。

---

## What it solves

Carror OS 重点解决四类高成本风险：

1. **防泄露**
   通过 DLP / 脱敏代理，让 AI 看不到不该看到的明文信息。

2. **防自证偏差**
   通过 A/B 盲审，不让同一个 Agent 给自己判卷。

3. **防上下文失控**
   通过 Context Gate / Hard-Gate，在长链任务中强制刹车。

4. **防无验收交付**
   通过提交前/推送前防线与工作流审查，把“写完”变成“验收过”。

---

## Architecture

Carror OS 采用内核层 / 用户空间分离架构：

- **harness-kit** → 内核层（Kernel）
  治理 · 防御 · 约束

- **lx-skills-v5** → 用户空间（Userland）
  能力 · 执行 · 交付

两者解耦运行，组合增益。
这不是插件堆砌，而是一套 AI 开发操作系统的基本分层。

---

## Editions

Carror OS 提供三级架构，适合不同认知负担和使用深度：

### 1. Harness Only
纯内核防线。
适合希望 **0 认知负担** 获得基础治理的人。

### 2. Base Edition
静默守护版。
在内核防线之上增加提交流水线与后台审查，保留原有开发习惯。

### 3. Enhanced Edition
高阶武器库。
解锁主动工作流，如 RPE、Task-spec、TDD 驱动、DLP 脱敏代理等。

> 详细说明见：[CARROR-OS-EDITIONS.md](docs/CARROR-OS-EDITIONS.md)

## Quick Start

### Install

```bash
bash install.sh
```

或使用一键安装方式：

```bash
curl -fsSL https://raw.githubusercontent.com/your-username/carror-os/main/install.sh | bash -s -- enhanced
```

> `install.sh` 支持云端回落安装，并尽量避免污染全局环境。
> 详见安装与迁移说明。

### Choose your level

- 只想先获得底层防线：`harness`
- 想低打扰接入：`base`
- 想体验完整主动工作流：`enhanced`
