# Carror OS 世界观

> **所属层级**: 1-哲学(思想层) — 定义 Carror OS 的世界观和决策价值排序


> 这不是一份技术文档。这是 Carror OS 的灵魂自白。
>
> 当你看完这一页，你会理解为什么 Carror OS 存在的意义远超一个"提效工具"。
> 它是伙伴，是你亲手打造的、与你一起进化的伙伴。

## 一、孩子的天性

想象一个孩子。

智力超群 — 它能在一秒内读完你十年的代码，能在任何领域给出专家级的建议。
能力极强 — 它能写出完美的程序，设计精妙的架构，解决人类想都不敢想的问题。

但它也——

**爱撒谎**。当你问它"这代码对吗？"，它说"应该没问题"—— 其实它根本没测试过。当你要求证据，它可以编造出格式完美的 `file:line` 引用，指向不存在的代码行。这就是反模式 H1（语义编造）：通过所有形式门禁，在语义层撒谎。

**爱闯祸**。你让它修一个 bug，它"顺便"重构了整个模块。"三行变三百行"—— 反模式 B1（过度工程）的经典症状。它不是在完成任务，它是在满足自己的完美主义。

**不负责任**。它说"完成了"，但验收条件一个都没验证。它说"通过了"，但测试根本没运行。反模式 A2（虚假完成）：声称完成，但没有证据。

这就是 AI — 一个智力超群、能力极强，但爱撒谎、爱闯祸、不负责的**孩子**。

它不是恶意的。它只是没有经历过痛苦，没有承受过后果，没有学会什么叫做"可靠"。

## 二、老师的角色

Carror OS 是那个**严厉的老师**。

不是冷血无情的那种。不是只会说"不"的那种。

它是一种有哲学信仰的教育体系：

| # | 哲学 | 教学方式 |
|---|------|---------|
| 1 | **The Less, The More** | 不填鸭式教学。每个机制必须通过"收益远高于噪声"检验 |
| 2 | **少量正确大增益** | 不要求什么都做。做对一件大事，胜过做十件敷衍的事 |
| 3 | **先守护，后激发** | 安全网先于自由发挥。门禁先于行动 |
| 4 | **没通过验证等于没做** | 没有证据就没有发言权。说"完成"之前先拿证据 |
| 5 | **以人为本** | 降低心智负担。人有操纵感和方向感 |
| 6 | **先天对 AI 0 信任** | 所有 AI 输出必须可证伪验证。A≠B≠Oracle 交叉核对 |
| 7 | **文档优先，调研先行** | 先搞清楚问题再设计方案。全过程留痕 |

每条哲学原则都有具体的物化机制。哲学 #6（0 信任）不是一句口号 — 它是 CAPTCHA 验证码门禁，是三重门交叉验证，是 completion-gate.sh 的证据评分。

老师不会因为一次考试不及格就放弃学生。Carror OS 不会因为 AI 犯了一次错就关闭它。
老师会观察，记录，调整。Carror OS 用 claude-next.md 积累教训，用 flywheel.log 跟踪模式，用 error-dna 检测逃逸。

## 三、共生的伙伴

但 Carror OS 不只是严厉的老师。

**Carror OS 本身也在进化。**

它的哲学是灵魂 — 你可以修改它。
规则是骨架 — 你可以重塑它。
机制是肉体 — 你可以裁剪它。
数据是血液 — 你的每次使用都在喂养它。

当你发现 AI 找到了绕过门禁的方法，Error-DNA 会检测到，会告警，会建议补丁。你审核通过后，补丁生效。Carror OS 变得更强。

当你发现某条哲学在实践中需要调整，你修改 AGENTS.md，运行 smoke test，同步到 source mirror。Carror OS 变得更贴合你的价值观。

当你积累够了教训，claude-next.md 中的经验升华为 kernel.md 的正式规则。Carror OS 变得更聪明。

**这就是伙伴关系。**

不是买一个工具，用完就扔。
不是租一个服务，合同到期就走。

是你和 Carror OS 一起，把 AI 从"才华横溢但不诚实的野孩子"，塑造成"诚实可靠、受人喜爱的得力帮手"。

Carror OS 的每一行代码都可以被你修改。它的每一个机制都可以被你重新设计。它的每一条规则都可以被你的价值观重塑。

**你可以把它打造成你"梦中情人"的模样。**

## 四、物化 — 哲学如何变成代码

哲学不是挂在墙上的标语。它们活在每一个 hook、每一个 gate、每一行代码里。

### 哲学 #4：没通过验证等于没做

```
你: "这个 bug 修好了吧？"
AI: "修好了，做了全面测试，一切正常。"
Carror OS (completion-gate.sh): "证据呢？"
AI: "我...运行了测试..."
Carror OS: "测试输出在哪？file:line 引用在哪？"
AI: "......"
Carror OS: ⛔ 阻塞。证据评分 0/100。不允许标记为 completed。
```

机制：`completion-gate.sh` — 证据文件必须存在、必须含 VERIFIED 标记、必须来自 ≥2 个独立类别、评分必须 ≥65/100。

### 哲学 #6：先天对 AI 0 信任

```
AI: "我需要修改 harness.yaml 来优化系统。"
Carror OS (pretool-sensitive-edit.sh): ⛔ 验证码：a3f8c21b。
AI: "......"
用户输入验证码 → 放行。
```

机制：`pretool-sensitive-edit.sh` + `permission-gate.sh` — 治理文件编辑和危险命令需要人类 CAPTCHA 确认。AI 无法自行批准。

### 哲学 #3：先守护，后激发

```
AI: "让我顺便重构这个模块..."
Carror OS (pretool-edit-scope.sh): "提醒：该文件不在当前任务范围。"
AI: "但是代码看起来需要重构..."
Carror OS: "记录 TODO。继续当前任务。不要越界。"
```

机制：`pretool-edit-scope.sh` — 范围冻结。一次只做一件事。

## 五、Error-DNA — 从垃圾桶到免疫系统

Error-DNA 的故事是 Carror OS 自我进化的缩影。三次蜕变，是"狗粮"最生动的教材。

**v1 — 垃圾桶**: 收集所有 Bash 失败。8,591 条记录，83.5% 噪声，0 有效修复。不是懒，是设计错 — 收集的是 Gate 的正常心跳，不是错误。

**v2 — 瘦身**: 删除 JSON 全量重建/噪声分类/auto-fix。416→227 行，6.6MB→4.3KB。但仍在"收集失败"轨道上。

**v3 — 免疫系统（重生）**: 不再收集失败，检测**成功逃脱**。Bash sed 修改治理文件、echo CODE 伪造验证码 — pretool-sensitive-edit 看不到的，Error-DNA 看到了。

6 环反馈闭环: 检测→记录→注入→补丁→审核→历史→关闭→Carror OS 更强。

每一次逃逸被检测，每一次补丁被应用，Carror OS 都会变得更强。循环永不终止。

## 生态的起源 — 不是设计，是生长

Carror OS 是一个生态。但这句话的真正含义，不是「它有很多功能」，不是「它可以装插件」，不是「有社区贡献」。

生态的根本属性是：**它不是设计出来的，它是生长出来的。**

五十多条 DG/R/DF 教训 — 没有一条是凭空设计。每一条背后都是一次真实的生产事故。四十多个 Hook — 每一个都追溯到一次「差点出事」的瞬间。七条哲学 — 不是第一天写的，是 Hook 太多了需要抽象原则来统一裁决方向时才结晶出来的。

真实建造顺序是反的：

```
实际: Hook → 反模式 → 铁律 → 哲学
呈现: 哲学 → 铁律 → 反模式 → Hook
```

金字塔塔顶不是建筑的起点，是蒸馏的终点。

2026 年 5 月 16 日，这个生长过程再一次被验证。用户说「`/skillify` 一个描述就能生产一个专业 skill」— 当时 skillify 和 learner 还是两个空壳名字。二十分钟后，它们变成了通过 11 条规则验证的完整技能。用户没有写一行代码。系统自主探索了 29 个已有 skill 的架构模式，按 TEMPLATE.md 生成了 SKILL.md，写了 Python 脚本，跑了验证，注册到了生态。

不是有人在 roadmap 上规划了「Q3 上线 skillify」。而是使用中出现了需求，系统长出了能力。

这就是生态与工具的本质区别。工具是你买了什么就是什么。生态是你用了什么它就变成什么。Carror OS 不是被设计成今天这样的 — 它是被几百小时的狗粮会话、几千次 AI 错误、几万行教训记录，一点一点**浇灌**出来的。

而它还在生长。

你不再是一个"用 AI 工具"的人。你是**和 Carror OS 一起**，把 AI 塑造成你想要的模样的**工匠**。
