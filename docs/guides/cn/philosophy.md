# Carror OS 哲学体系

> **所属层级**: 1-哲学(思想层) — 7 条哲学原则详解 + 优先级裁决规则


> Carror OS 不是一个工具集 — 它是一套有性格的 AI 操作系统。性格由哲学定义。

---

## 为什么 AI 需要哲学？

每个人都有自己的哲学 — 那是经历过无数次决策后沉淀下来的价值排序。同一个人面对同一件事，多次决策结果一致，因为他的哲学在背后统一调度。

AI 也一样。没有哲学的 AI 助手每次决策都是掷骰子 — 这次严格下次宽松，这次激进下次保守。有了哲学，AI 的行为可预测、可信任、可调校。

**Carror OS 的 7 条哲学 = 它的性格。** 这不是贴在墙上的口号，而是物化为 40+ 个自动化门禁（hook）的执行代码。每一条哲学都有具体的机制来实现它，每一个机制都可以追溯到它服务的哲学。

---

## 7 条哲学

| # | 哲学 | 含义 | 日常体现 |
|---|------|------|---------|
| 4 | **没通过验证等于没做** | 没有证据的"完成了"等于没做 | 每次标记完成前强制验证，软完成语被拦截 |
| 6 | **先天对 AI 0 信任** | AI 的每一句话都需要独立验证 | 技术断言必须有 file:line，自创指标被拒绝 |
| 3 | **先守护，后激发** | 安全网比能力更重要 | 危险命令需要人类批准，敏感文件编辑需要验证码 |
| 7 | **文档优先，调研先行** | 先理解再行动，全程留痕 | 复杂任务强制先出方案，Oracle 审核后执行，关键节点 Meta-Oracle 最后守门 |
| 5 | **以人为本** | 降低心智负担，选择优于输入 | 菜单式交互、图标提示、方向感反馈 |
| 2 | **少量正确大增益** | 做对的事而不是做多的事 | 不顺手修无关 bug，修复 3 轮上限 |
| 1 | **The Less, The More** | 隐藏复杂性，渐进式披露 | 大量机制静默运行，只在必要时提醒 |

### 优先级链条

当哲学冲突时（如"快速修复" vs "先写文档"），按固定优先级裁决：

```
验证(#4) > 不信任(#6) > 安全(#3) > 文档(#7) > 体验(#5) > 聚焦(#2) > 简洁(#1)
```

---

## 它不是硬编码的规则

**Carror OS 是一套可重新编程的价值排序系统。**

当你发现它的行为不符合预期时，不需要绕过机制，不需要修改代码 — **调整哲学优先级即可**：

- 更信任 AI？降低 #6 的优先级
- 追求效率？提升 #2 的优先级
- 需要更强控制感？提升 #5 的优先级
- 甚至可以替换某条哲学为你自己的原则

调整后，所有 40+ 个 hook 的行为自动跟随变化。把它塑造成你的「梦中情人」— 让它像你一样思考。

## 机制→哲学 追溯

| 你看到的现象 | 背后的机制 | 服务的哲学 |
|-------------|-----------|-----------|
| 说"完成了"被拦截 | completion-gate 证据评分 | #4 没通过验证等于没做 |
| git push 前弹出验证码 | permission-gate CAPTCHA | #6 先天对 AI 0 信任 |
| 上下文高了不让写代码 | context-guard 硬阻断 | #3 先守护后激发 |
| 修 bug 被要求先读代码 | edit-guard Read-before-Edit | #6 + #3 |
| 输出太随意被提醒 | posttool-format-gate | #5 以人为本 |
| 同一错误重试 3 次被停 | pretool-retry-check.sh（重试预算上限） | #2 少量正确大增益 |
| 新建会话自动加载上下文 | inject-project-knowledge | #7 文档优先 |
