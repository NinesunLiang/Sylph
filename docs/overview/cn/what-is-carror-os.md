# Carror OS 是什么？

> **Carror OS 是 AI 原生开发者操作系统。**
>
> 它将 AI 编程从"凭感觉"转变为"凭证据"。

---

## 问题所在

当你用 AI 写代码时，同样的灾难一遍又一遍发生：

- 长对话质量下降 — 超过 20 轮后，AI 开始幻觉并删除正常工作的代码
- 公司机密泄露 — AI 读取 `.env` 文件并向外部服务器发送凭据
- AI 自我审查自己的作品并宣称"完美" — 因为它已经对自己的错误做出了承诺
- Token 浪费在成千上万行的规则上，AI 既无法遵循也无法记住

Carror OS 诞生于六个月的现实世界 AI 编程失败经验 — 不是理论架构，而是战场生存。

## 四大核心能力

| 能力 | 功能 |
| :--- | :--- |
| **门禁（物理拦截）** | 在危险 AI 行为执行前，在文件系统级别物理阻断。不是"请不要"——而是"你做不到" |
| **上下文控制（记忆保险丝）** | 监控真实 Token 消耗。在 80% 时物理终止会话，防止幻觉驱动的破坏 |
| **审计追踪（取证证据）** | 通过 SHA256 完整性记录每一个 AI 行为。每个决策都可追溯、可重放、可验证 |
| **工作流（规范管线）** | 用结构化、证据门禁的开发流程（RPE、多智能体盲审）取代临时 AI 指令 |

## 它有什么不同？

Carror OS **不依赖提示工程**。它不会把你的规则放到系统提示中然后指望 AI 遵循。

相反，它在 AI 和你的文件系统之间放置**物理钩子** — 通过 Claude Code 的 Hook 协议在应用层实现，在危险工具调用执行前将其阻断。AI 无法通过巧言令色绕过这些钩子。
