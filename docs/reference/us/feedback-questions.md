# External Tester Feedback Questions

> Use these questions when onboarding external testers to collect structured, actionable feedback.
> Ask these after the tester has completed the First 10 Minutes guide and tried Carror OS on their own.

---

## Core Comprehension

1. **Can you describe what Carror OS is in 60 seconds?**
   - Listen for: clarity of value proposition, core concept retention

2. **How is it different from Cursor / Claude Code?**
   - Listen for: understands governance layer vs code generation tool

3. **Which feature would you most want to try first?**
   - Listen for: which pain point resonates most

---

## Onboarding Experience

4. **Where did the installation get stuck or feel confusing?**
   - Critical for identifying friction points in setup

5. **Did the first "aha moment" happen? When?**
   - Listen for: did the completion gate trigger successfully?
   - If no aha moment — what needs to change?

---

## Trust & Concerns

6. **Would you use Carror OS in a real production repository?**
   - If no → probe for trust, complexity, or fear concerns

7. **What worries you about using this system?**
   - Listen for: safety concerns, lock-in, complexity, performance

8. **If your team were to adopt this, what documentation would you need most?**
   - Prioritizes documentation gaps

---

## Commercial

9. **Would you (or your team) pay for a workflow review or team pilot? Why or why not?**
   - Validates pricing hypotheses and uncovers objections

10. **Is there anything that would make this a "must-have" instead of "nice-to-have"?**
    - Identifies the gap between interest and urgency
