# 18. 你需要准备 10 个质疑的标准回答

发布后别人一定会问这些。

## 1. 和 Cursor 有什么区别？

> Cursor is an AI-native IDE. Carror OS is a local-first governance and workflow layer for AI coding sessions. It focuses on gates, context control, audit trails, and evidence-before-done workflows.

中文：

> Cursor 是 AI IDE。Carror OS 是 AI 编程治理与工作流层，重点不是补全体验，而是门禁、上下文控制、审计追踪和完成前证据。

---

## 2. 和 Claude Code 有什么区别？

> Carror OS is not a replacement for Claude Code. It is a harness layer on top of Claude Code, adding governance, audit, feature registry, and structured workflows.

---

## 3. 是不是太重？

> 如果你只是做一次性小修改，它可能偏重。Carror OS 主要面向真实生产式 AI coding session：需要边界、审计、验收和可恢复性的场景。

---

## 4. token 到底省多少？

> 我们正在用 benchmark 谨慎测量。Progressive disclosure 的目标是减少不必要上下文加载，但具体节省比例必须以可复现 benchmark 为准。

---

## 5. 这是不是 prompt engineering？

> 不是。Carror OS 不是一组 prompt，而是本地 harness 层，包括 hooks、gate、registry、audit、状态文件和工作流约束。

---

## 6. 会不会影响效率？

> 会增加少量关键节点摩擦，但目标是减少返工、假完成和失控修改。对于生产式任务，正确的摩擦是收益，不是成本。

---

## 7. 适合谁？

> 适合 Claude Code / AI coding 重度用户、独立开发者、小团队技术负责人、安全敏感团队，以及希望把 AI coding 放进真实交付流程的人。

---

## 8. 不适合谁？

> 如果你只需要快速补全、一次性小改动、零配置体验，Carror OS 可能不是第一选择。

---

## 9. 是否生产可用？

> 核心能力正在真实生产流程中 dogfooding。部分高级能力仍为 experimental，并在文档中明确标注。
