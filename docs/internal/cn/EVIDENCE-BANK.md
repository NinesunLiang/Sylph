## 指南

1. 每条证据条目必须包含捕获日期、上下文和文件路径。
2. 截图应放入 `docs/internal/assets/` 或链接到外部托管。
3. 存储前对敏感信息（仓库名称、用户名、密钥、内部路径）进行脱敏处理。
4. 用证据级别（内部门禁按 AGENTS.md 的 L1-L4，或面向外部受众的 C1-C5）标记每条条目。

## 资产类型

- **截图**：门禁触发、lx-status 面板、审计日志、安装流程
- **日志**：终端输出、error-dna 记录、飞轮事件
- **基准测试**：加载基准测试报告、token 对比表
- **狗粮测试**：带有前后对比的真实任务完成记录
- **测试结果**：验收测试报告、手动测试日志
- **外部来源**：用户反馈、评论引语、社区回应

## 脱敏检查清单

在发布任何证据资产前，请检查：
- [ ] 仓库名称已移除或匿名化
- [ ] 用户名已移除
- [ ] 密钥 / Token / 密码已移除
- [ ] 不存在客户或用户数据
- [ ] 内部文件路径已移除
- [ ] 无敏感 IP 或专有代码可见
