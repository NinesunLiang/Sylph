## 0.1 四项提分是否成立

### 裁决：**有条件成立**

「四项」若指生命周期/handoff 相关提分（C2 上下文完整、E8 抗遗忘、抗衰减、registry 诚实/生命周期闭合），PKG-C 的**机制目标**可支撑 7→9 量级，但 **文档自报分 ≠ 成立**。

| 提分项 | 成立条件（机械） | 不成立信号 |
|--------|------------------|------------|
| C2 计数/上下文 | `handoff.written == claimed == len(items)`；A5/A11 绿 | 仍口头「0/0」或 md Progress 当真相源 |
| E8 抗遗忘 | PreCompact 必有 snapshot + sha256 回读一致；失败 exit 2 | PreCompact exit 0 吞失败 |
| 抗衰减 / 闭合 | SubagentStop 幂等 + SessionEnd/Stop seal → `mode=idle` | Stop 拆掉 flywheel 或无 seal |
| registry 诚实 | `auto-snapshot` = zombie + `enabled_by_default: false`；新 lifecycle 条存在 | 双 snapshot 写路径 / 谎称默认启用 |

**哲学**：提分属 **验证** 与 **文档** 交界——**验证优先**。  
**结论句**：  
> 四项提分 **仅在 A0–A12（或等价 jq+hook 门）全绿时成立**；任一门红，该项提分与总分中的对应浮点 **一并作废**，不得用「分数表已经写了」反压验收。

**对整合器**：Room 表若已写「+1.x」，必须回溯挂证据哈希/命令日志；否则一律打回「未验证提分」。

## 0.2 E7 是否接受 hint-only 终态

### 裁决：**不接受 hint-only 作为永久终态**

E7（oracle / 验证门「仅 hint、不 BLOCK」类半开状态）与哲学主链冲突：

```
验证 > 零信任
⇒ 未证实通过 ≠ 可通过
⇒ 默示放行（hint-only 当终态）= 零信任违约
```

| 选项 | 可否作为 R 收口终态 | 理由 |
|------|---------------------|------|
| hint-only 永久默认 | **否** | 可绕过证据门禁，整条验证链塌陷 |
| BLOCK 直至证据 | **是（工程默认）** | 服务 验证>零信任>守护 |
| 人类独占、书面、不可逆豁免 | **是（程序挂起）** | 「人类独占不可逆裁决」；须落盘裁判记录（时间、人、范围、回滚条件） |
| 代码里写死「永远 hint」却宣称验证链完工 | **否；整项 E7 不计分** | 文档骗分 |

**边界（本包自觉）**：E7 属 PKG-A/B / 人类域；**PKG-C 不施工、不搭新层「修」E7**。  
**委员会立场（Grok 票）**：  
> **反对** 将 hint-only 写进「终态 DONE」。  
> **同意** 采购此时人类裁决：**要么** 收成 fail-closed BLOCK，**要么** 单独开「E7-HUMAN-WAIVER」档案并 **E7 分项打折或不计入 8.51 的验证分母**。  
> 把豁免伪装成「系统已验证」→ **否决**。

## 0.3 内置安全 7 是否豁免

### 裁决：**豁免——成立**

「内置安全 7」按守护语义之间：平台/内核已装安全闸（权限、密钥、危险工具默认拒、隐私红线等），**不得为抬综合分而关闭、旁路、降级或「评分优化豁免口」**。

| 维度 | 裁定 |
|------|------|
| 是否可改代码关掉换分 | **禁止** |
| 是否计入「未完成优化债」逼 R6 | **否，豁免** |
| 与「少」冲突时 | **守护 > 增益 > 少** → 保留安全 |
| 与提分冲突时 | **提分让路** |

**哲学落点**：守护 > 文档/增益。  
**一句话**：  
> 内置安全相关项 **固定豁免**；任何「为冲 9.x 关掉安全」的提案 **直接作废**，不进 R6 施工队列。
